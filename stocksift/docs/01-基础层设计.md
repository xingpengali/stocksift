# 基础层设计文档

## 1. 概述

基础层为整个应用提供基础设施支持，包括配置管理、日志系统、工具函数等。该层不依赖业务逻辑，可被所有上层模块调用。

## 2. 配置管理模块 (config/)

### 2.1 功能职责
- 应用配置的读取、存储、更新
- 支持多环境配置（开发/生产）
- 配置变更通知机制

### 2.2 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| 配置管理器 | `settings.py` | 配置的CRUD操作 |
| 常量定义 | `constants.py` | 全局常量、枚举 |

### 2.3 配置分类

```
config/
├── settings.py          # 配置管理类
│   ├── Settings         # 主配置类
│   ├── load()           # 加载配置
│   ├── save()           # 保存配置
│   ├── get(key)         # 获取配置项
│   └── set(key, value)  # 设置配置项
│
└── constants.py         # 常量定义
    ├── 市场常量（交易所、板块）
    ├── 数据周期常量
    ├── 运算符常量
    └── 枚举类定义
```

### 2.4 配置存储格式

```json
{
  "app": {
    "window_width": 1400,
    "window_height": 900,
    "theme": "light",
    "language": "zh_CN"
  },
  "data_source": {
    "tushare_token": "xxx",
    "priority": ["tushare", "baostock"]
  },
  "user_preferences": {
    "refresh_interval": 5,
    "alert_sound": true
  }
}
```

## 3. 日志模块 (utils/logger)

### 3.1 功能职责
- 应用日志记录与管理
- 多级别日志输出
- 日志文件轮转

### 3.2 核心接口

```python
# 初始化日志系统
setup_logging(
    log_dir: str,
    console_level: str = "INFO",
    file_level: str = "DEBUG"
)

# 获取日志记录器
logger = get_logger(__name__)
logger.info("消息")
logger.error("错误", exc_info=True)
```

### 3.3 日志格式

```
[2024-01-15 10:30:25] [INFO] [module.name:42] 日志消息内容
```

### 3.4 日志文件管理

```
logs/
├── stocksift.log          # 当前日志
├── stocksift.log.1        # 轮转备份
└── error.log              # 错误日志单独存储
```

## 4. 工具函数模块 (utils/)

### 4.1 功能分类

| 模块 | 功能 |
|------|------|
| `helpers.py` | 通用工具函数（日期、数值格式化） |
| `validators.py` | 数据验证、输入校验 |
| `decorators.py` | 通用装饰器（重试、计时、缓存） |
| `cache.py` | 内存缓存管理 |
| `event_bus.py` | 组件间事件通信 |
| `exporter.py` | Excel/CSV导出 |

### 4.2 缓存设计

```python
class Cache:
    def get(key: str) -> Any
    def set(key: str, value: Any, ttl: int = 3600)
    def delete(key: str)
    def clear()
```

### 4.3 事件总线

```python
event_bus = EventBus()

# 订阅事件
event_bus.subscribe("quote_updated", handler)

# 发布事件
event_bus.publish("quote_updated", data)
```

## 5. 模块依赖关系

```
config/constants ──┐
                   ├──► 所有上层模块
config/settings ───┤
utils/logger ──────┤
utils/helpers ─────┘
```

## 6. 关键设计决策

| 决策 | 说明 |
|------|------|
| SQLite配置存储 | 简单场景使用JSON，便于手动编辑 |
| 单例模式 | Settings和Logger使用单例，全局唯一 |
| 异步日志 | 避免日志IO阻塞主线程 |
| LRU缓存 | 内存缓存使用LRU策略，防止内存溢出 |
